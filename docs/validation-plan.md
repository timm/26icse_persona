# Validation Plan

How to show the composite 7D persona model (GenderMag cognitive facets + cocomo.tex organizational dimensions, from `background.md`) is *insightful* — i.e., adds predictive power over GenderMag alone and matters for tool evaluation.

Four tests. Each must pass for the claim to hold. Order matters: the later tests are only meaningful if the earlier ones pass.

## Test 1 — Distinguishability

**Question.** Are the new axes (Control, Now) behaviorally detectable in persona output, or do they collapse onto the cognitive ones?

**Hypothesis.** An independent classifier can recover Control and Now at above chance from persona-generated text.

**Method.** Generate persona responses on a shared stimulus. Use LLM-as-Judge blind classification (SE4AI §4.3.4 protocol) to recover facet values from text.

**Design.** For each new axis (Control, Now):
- Hold all other facets fixed at a canonical mid-value.
- Vary the target axis across its range (e.g., Control ∈ {0, 2, 4}).
- Generate N responses per level.
- Judge identifies level from text.

**Measures.** Classification accuracy; Krippendorff α, Cohen κ for inter-judge reliability (SE4AI: α ≥ 0.80, κ ≥ 0.61).

**Pass.** Accuracy significantly > chance (e.g., > 50% for 3-level classification, chance = 33.3%). Inter-judge reliability meets SE4AI thresholds.

**Fail.** Classifier cannot recover the axis. Drop it or redefine it.

## Test 2 — Non-redundancy

**Question.** Are the new axes predictable from GenderMag facets? If so, they add nothing.

**Hypothesis.** Control and Now are not reliably predictable from {motivation, info-style, self-efficacy, risk, learning-style}.

**Method.** Generate a grid of responses varying all 7 facets. Fit a regression / classifier predicting each new axis from the 5 GenderMag axes. Report R² (continuous) or balanced accuracy (discrete).

**Design.** Latin hypercube or full factorial over the 7D space at 3 levels each, responses generated by one LLM.

**Measures.** R² or accuracy for predicting Control from GM facets; same for Now. Cross-validated.

**Pass.** R² < 0.3 (or classifier barely beats chance). New axes carry information the old ones don't.

**Fail.** R² > 0.7. New axes are redundant; the paper's claim collapses.

## Test 3 — Predictive power (the core ablation)

**Question.** Do the new axes change acceptance/ranking decisions compared with GenderMag alone?

**Hypothesis.** Composite 7D personas reach decisions that diverge from GenderMag-only personas on cases where the GenderMag model would predict agreement.

**Method.** Three-scheme ablation on the same SE task:

| Scheme | Personas | What it tests |
|---|---|---|
| A | GenderMag only (3 or 9, SE4AI-style) | baseline cognitive-only |
| B | cocomo.tex only (8 canonical groups or weighted mixes) | positional-only |
| C | Composite 7D (cross of A and B) | combined |

**Design.**
- Select k SE artifacts (see *Prerequisite* below).
- Each persona in each scheme produces a structured JSON response per SE4AI §4.3.2: `accepts_recommendation`, `clarity_score`, `sufficiency_score`, `preferred_rank`, free-text reasoning.
- Aggregate per-scheme decisions.
- Compute pairwise divergence: fraction of artifacts on which scheme C diverges from A, and from B.

**Measures.**
- **Divergence rate**: fraction of cases where schemes disagree on accept/reject.
- **Rank correlation**: Kendall τ between scheme rankings.
- **Scott-Knott clustering** of decisions (2105.12195 methodology) — are schemes in distinct clusters?

**Pass.** Scheme C diverges from A significantly (e.g., > 20% of cases), and in a non-random direction traceable to a specific new axis (Control or Now).

**Rhetorical jackpot.** Find a case where A predicts agreement across its 3–9 personas, but C reveals internal disagreement driven by Control or Now. One clean example is a quote-worthy figure for the paper.

**Fail.** A and C agree > 90% of the time. The new axes are real (Test 1, 2) but inert — they don't change decisions. Might still be worth a smaller paper, not a main-track contribution.

## Test 4 — Actionability

**Question.** Do the new axes lead to *different tool recommendations*, per the 2009 values paper standard?

**Hypothesis.** Running the stakeholder-pain calculus from `cocomo.tex` on the same SE recommendation under different persona mixes yields different chosen actions.

**Method.**
- Pick an SE tool with a real, parameterizable recommendation (e.g., microservice split granularity, bug-fix ranking, Pareto-front tree choice).
- Define 3–5 organizational personas as weighted mixes of canonical cocomo groups (e.g., startup = 1:2:2 customers:devs:execs; regulated-legacy = 2:1:1:3 users:devs:QA:regulators).
- Apply the pain calculus: `score(a) = w₁·change + w₂·effort + w₃·risk − w₄·value`, constrained by `pain(g,a) ≤ τ_g`.
- Report the top action per org mix.

**Measures.**
- **Action divergence**: do different org mixes select different actions?
- **Regret**: if you apply the exec-preferred action in an autonomy-preferring org, how bad is the pain penalty?
- Confirm the 2009 pattern: *"if one value function approves X, the other approves not X."*

**Pass.** Multiple org mixes → multiple chosen actions, with documentable pain regret for mismatched pairings.

**Fail.** All org mixes converge on the same action. Means the task's objective function dominates; the stakeholder calculus is not load-bearing for this task.

## Design decisions to lock before running

1. **LLM-only vs. LLM + human spot-check.** SE4AI goes all-LLM, citing Schuller et al. 2024 (CHI). That claim is load-bearing and still contested. Option: replicate SE4AI's approach for Tests 1–3, but do a small (n=5–10) human spot-check specifically for the *new* axes (Control, Now), to avoid inheriting a single-paper assumption. Human validation of only the novel dimensions is defensible and publishable regardless of outcome.
2. **Persona generation strategy.** Fixed grid (for ablation clarity) vs. weighted mixes from `cocomo.tex` (for real-world realism). Recommend: fixed grid for Tests 1–2, weighted mixes for Tests 3–4.
3. **Judge models.** SE4AI uses Claude Sonnet 4.5 + Llama 3.1 70B. Match for direct comparability; or swap in Claude Opus 4.7 + a second frontier model for stronger judges.
4. **Statistical tests.** Scott-Knott (2105.12195 pattern) for clustering; Wilcoxon rank-sum with Bonferroni for pairwise (SE4AI §4.1 pattern). Report effect sizes (Cohen's d, rank-biserial correlation).

## Prerequisite: pick the SE task

None of the above runs without a concrete SE artifact to evaluate stakeholders *on*. Candidates:

- **MOOT optimization trees** (SE4AI's task). Already calibrated, Pareto fronts have role-relevant trade-offs, MOOT has 120+ tasks to scale to.
- **Microservice identification** (Carvalho 2024's task, `toMicroservices`). Real industrial ground truth, direct continuity with the 2024 TSE paper.
- **Defect-fix ranking** (continuity with Menzies et al. defect prediction line).
- **Cost-estimation / architecture recommendation from COCOMO** (direct use of `cocomo.tex`'s driver model).

Any of these work. MOOT has the lowest friction — infrastructure, benchmarks, validation already done by SE4AI.
